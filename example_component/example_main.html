AngularJS Demo

AngularJS uses the Model-View-Control Architecture to implement its design and functionality. In AngularJS, a HTML template provides a View of a data Model. This interaction is overseen by a Controller.

In other words, Views "plug" into Controllers so they can access and present the Models to the user.

AngularJS Functionality

1: Dependency Injection

Here is an HTML element containing the ng-controller directive.

The ng-controller' directive allows you to inject data from Models into the View by accessing the Controller. If you click F12 on your keyboard and open the Network tab on the Developer console, you will see a script example_controller.js being served up to your browser. This script contains the data being injected into the View. Take a look at the Response in the Developer console to see how this accomplished. Notice the name of function in the Response exactly equals the name of this HTML element. This is no coincidence! This is how AngularJS knows to inject the Controller into this element!

Technically, it is the $scope passed an argument into the controller that opens a channel through the Controller from the Model to the View

We use the Controller data injection inside of another Angular directive, the ng-repeat directive to illustrate the potential use for this type of functionality.

2: Data Binding

The ng-model directive provides a method of data-binding. A data-binding in Angular is a special type of dependency-injection. Underneath the hood, the ng-model uses a special type of Controller to create a new data object on the $scope object. Recall the $scope provides a means of communication between the Controller and the View.
